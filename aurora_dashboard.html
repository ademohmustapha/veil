Why AURORA Wins

Genuinely Autonomous Co-Evolution
The only platform with a genetic algorithm that evolves its own detection rules each generation — discovering attack patterns no human researcher anticipated. Rules improve continuously across organisations without any manual updates.
Verified Quantum-Safe KEM Roundtrip
Full ML-KEM (Kyber-512/FIPS 203) implementation with real NTT-domain polynomial arithmetic, CSPRNG noise sampling, proper compression/decompression, and verified encap/decap round-trip (10/10 iterations). Hybrid X25519 + ML-KEM: both barriers must be broken simultaneously.
Causal Bayesian Threat Forecasting
Uses Noisy-OR Bayesian networks with 20 precursor signals to predict attacks 6–72 hours before materialisation. Monte Carlo forward simulation with confidence intervals — not rules-based alerting.
3-Pillar Autonomous Ethical Governance
Every AI decision passes through Deontological (hard rules) → Consequentialist (outcome analysis) → Proportionality checks. Human override is always preserved. Bias detection runs on every containment action.
Differential-Privacy Federated Intelligence
Laplace mechanism (ε-DP) adds calibrated noise before sharing threat intelligence across organisations. Your internal risk scores are never exposed. The network learns collectively without any participant seeing individual data.
Self-Hardened Against Its Own Threats
Rate limiting, SQL/command injection prevention, path traversal blocking, integrity self-hash baseline, and tamper detection — all built in. A security tool that is itself insecure is a liability. AURORA protects itself first.

ENTERPRISE CONNECTIVITY

Identity Integration Layer
Connect AURORA to your organisation's identity infrastructure

AURORA's Identity Integration Layer bridges the platform with your existing identity and HR systems. Once configured, AURORA automatically knows your organisation, every employee, their ID, department, role, and risk profile — with zero manual input required after setup.

PRIORITY 1 — PREFERRED
SSO / OAuth2 / OIDC / SAML 2.0
Full browser-based authentication flow. Works with Okta, Azure AD, Google Workspace, Ping Identity, Shibboleth, ADFS. User IDs and org IDs are pulled automatically from the identity token — no typing, no manual entry.
Okta Azure AD Google Workspace SAML 2.0
PRIORITY 2
LDAP / Active Directory
Auto-detects the current OS user and resolves their LDAP record — including department, role, and employee ID — without any additional login. Connects to any LDAP v3 or AD domain controller.
Active Directory OpenLDAP FreeIPA
PRIORITY 3
Org Config + Employee Roster
Register your organisation once in ~/.aurora/org_config.json. Import all employees in bulk via CSV or JSON — with support for custom column naming. IDs auto-populate from that point forward.
CSV Import JSON Import HR System Feed
PRIORITY 4 — FALLBACK
Environment Variables
Set AURORA_USER_ID and AURORA_ORG_ID as environment variables for CI/CD pipelines, containers, and automated deployments where interactive login is not possible.
Docker CI/CD Kubernetes

~/.aurora/identity.json — Identity Provider Configuration 
// Configure your preferred identity backend once. AURORA handles everything else.
{
  "active_backend": "auto",  // auto | sso | ldap | org_config | env

  "sso": {
    "provider":      "okta",
    "issuer_url":    "https://your-tenant.okta.com/oauth2/default",
    "client_id":     "${AURORA_SSO_CLIENT_ID}",
    "client_secret": "${AURORA_SSO_CLIENT_SECRET}"
  },

  "ldap": {
    "server":   "ldap://dc.corp.example.com:389",
    "base_dn":  "dc=corp,dc=example,dc=com",
    "bind_dn":  "cn=aurora-svc,ou=service-accounts,...",
    "auto_detect_current_user": true
  }
}
